# EdgeLink UI 界面实现文档

## 一、项目概述

### 1.1 项目背景

EdgeLink UI 是一个物联网设备管理平台的前端界面，主要用于管理物模型、产品和设备。本文档基于 `ui_desgin/ui.html` 原型设计，详细阐述如何使用 Ant Design 组件库结合 React 框架实现该界面。

### 1.2 技术栈选型

| 技术项   | 选型             | 选型依据               |
| ----- | -------------- | ------------------ |
| UI 框架 | Ant Design 5.x | 企业级组件库，组件丰富，设计规范完善 |
| 前端框架  | React 18       | 组件化开发，生态成熟         |
| 状态管理  | Zustand        | 轻量级状态管理，适合中大型项目    |
| css   | less           | <br />             |
| 路由管理  | React Router 6 | 声明式路由，支持嵌套布局       |
| 构建工具  | Vite           | 快速热更新，开发体验好        |
| 类型支持  | TypeScript     | 类型安全，代码可维护性强       |

## 二、整体架构设计

### 2.1 布局结构

界面采用经典的左侧导航 + 右侧内容区布局，整体结构如下：

```
┌─────────────────────────────────────────────────────────┐
│                      顶部标题栏                           │
├──────────┬──────────────────────────────────────────────┤
│          │                                              │
│   侧边   │                                              │
│   导航   │                 内容区域                      │
│   菜单   │                                              │
│          │                                              │
│          │                                              │
└──────────┴──────────────────────────────────────────────┘
```

### 2.2 页面路由规划

| 路由路径           | 页面名称  | 功能描述          |
| -------------- | ----- | ------------- |
| `/workbench`   | 工作台   | 数据概览、快捷入口     |
| `/tsl`         | 物模型管理 | 物模型列表、CRUD 操作 |
| `/products`    | 产品管理  | 产品列表、绑定物模型    |
| `/devices`     | 设备管理  | 设备列表、设备详情     |
| `/devices/:id` | 设备详情  | 单个设备的详细信息     |

## 三、组件拆分规划

### 3.1 布局组件

#### 3.1.1 MainLayout（主布局）

**职责**：整体页面框架，包含侧边导航和内容区域

**子组件**：

- `Sidebar`：侧边导航菜单
- `Header`：顶部标题栏
- `ContentArea`：内容区域容器

**Ant Design 组件映射**：

- 使用 `Layout` 组件作为主框架
- 使用 `Layout.Sider` 实现侧边栏
- 使用 `Layout.Header` 实现顶部栏
- 使用 `Layout.Content` 实现内容区

#### 3.1.2 Sidebar（侧边导航）

**职责**：展示导航菜单，支持页面切换

**设计要点**：

- 固定宽度（约 200px）
- 深色背景主题
- 菜单项包含图标和文字
- 当前页面高亮显示

**Ant Design 组件映射**：

- 使用 `Menu` 组件实现导航
- 设置 `theme="dark"` 深色主题
- 设置 `mode="inline"` 内联模式

### 3.2 通用组件

#### 3.2.1 PageHeader（页面标题）

**职责**：展示页面标题和操作按钮

**设计要点**：

- 左侧显示标题
- 右侧显示主要操作按钮

**Ant Design 组件映射**：

- 使用 `PageHeader` 或自定义 Flex 布局
- 使用 `Button` 组件实现操作按钮

#### 3.2.2 SearchBar（搜索栏）

**职责**：提供搜索和筛选功能

**设计要点**：

- 搜索输入框
- 筛选下拉框
- 搜索按钮

**Ant Design 组件映射**：

- 使用 `Input.Search` 实现搜索框
- 使用 `Select` 实现筛选下拉

#### 3.2.3 DataTable（数据表格）

**职责**：展示列表数据，支持排序、筛选

**Ant Design 组件映射**：

- 使用 `Table` 组件
- 配置 `columns` 定义列结构
- 配置 `dataSource` 绑定数据

#### 3.2.4 StatusTag（状态标签）

**职责**：展示设备在线/离线状态

**设计要点**：

- 在线：绿色圆点 + "在线"
- 离线：灰色圆点 + "离线"
- 空闲：蓝色圆点 + "空闲"

**Ant Design 组件映射**：

- 使用 `Tag` 组件
- 使用 `Badge` 组件显示圆点

#### 3.2.5 CodeTag（代码标签）

**职责**：展示标识符、Key 等代码内容

**设计要点**：

- 等宽字体
- 浅灰背景
- 圆角边框

**Ant Design 组件映射**：

- 使用 `Tag` 组件自定义样式

### 3.3 业务组件

#### 3.3.1 TSLList（物模型列表）

**职责**：展示物模型卡片列表

**设计要点**：

- 卡片式布局
- 每个卡片显示图标、名称、描述、属性数量
- 支持编辑、删除操作

**Ant Design 组件映射**：

- 使用 `Card` 组件展示单个物模型
- 使用 `List` 或 `Row/Col` 实现网格布局

#### 3.3.2 DeviceList（设备列表）

**职责**：展示设备表格数据

**设计要点**：

- 表格列：设备名称、产品、描述、创建/更新时间、状态、操作
- 操作列：详情、编辑、删除

**Ant Design 组件映射**：

- 使用 `Table` 组件

#### 3.3.3 DeviceDetail（设备详情）

**职责**：展示单个设备的详细信息

**设计要点**：

- 顶部：设备基本信息
- 中部：通信地址列表
- 底部：设备属性卡片

**Ant Design 组件映射**：

- 使用 `Card` 组件分区展示
- 使用 `Descriptions` 组件展示基本信息
- 使用 `Grid` 布局展示属性卡片

#### 3.3.4 PropertyCard（属性卡片）

**职责**：展示单个属性信息

**设计要点**：

- 属性名称和值
- 数据类型标签
- 单位显示
- 点击查看历史数据

**Ant Design 组件映射**：

- 使用 `Card` 组件
- 使用 `Statistic` 组件展示数值

### 3.4 弹窗组件

#### 3.4.1 AddTSLModal（添加物模型弹窗）

**职责**：分步添加物模型

**设计要点**：

- 第一步：配置基础信息（名称、标识符、描述、图标）
- 第二步：配置功能（属性、功能、事件）

**Ant Design 组件映射**：

- 使用 `Modal` 组件
- 使用 `Steps` 组件实现步骤导航
- 使用 `Form` 组件实现表单

#### 3.4.2 IconPicker（图标选择器）

**职责**：选择或上传物模型图标

**设计要点**：

- 左侧：上传图标区域
- 中间：预设图标选择（10个）
- 右侧：背景色选择（10种颜色）

**Ant Design 组件映射**：

- 使用 `Upload` 组件实现上传
- 使用 `Radio.Group` 实现图标选择
- 自定义颜色选择器组件

#### 3.4.3 InlineEditTable（内联编辑表格）

**职责**：在表格中直接编辑属性/功能/事件

**设计要点**：

- 点击底部加号添加新行
- 新行显示输入框和下拉框
- 确认/取消操作

**Ant Design 组件映射**：

- 使用 `Table` 组件
- 结合 `Form` 实现内联编辑

## 五、交互逻辑说明

### 5.1 导航切换

- 点击侧边菜单项，切换到对应页面
- 当前页面菜单项高亮显示
- 使用 React Router 进行路由管理

### 5.2 表格操作

- **查看**：跳转到详情页或打开查看弹窗
- **编辑**：打开编辑弹窗，预填充当前数据
- **删除**：弹出确认框，确认后删除

### 5.3 表单提交

- 必填字段验证
- 格式验证（如标识符只允许字母、数字、下划线）
- 提交成功后关闭弹窗，刷新列表

### 5.4 分步表单（添加物模型）

- 步骤导航显示当前步骤
- 第一步完成验证后才能进入第二步
- 支持上一步/下一步切换
- 最终提交时收集所有步骤数据

### 5.5 内联编辑表格

- 点击底部加号，插入新行
- 新行显示输入框和下拉框
- 点击确定保存数据，点击取消删除新行
- 保存后更新 Tab 标签的数量显示

### 5.6 图标选择器

- 点击预设图标，选中并更新预览
- 点击颜色选项，更新图标颜色
- 上传自定义图标，替换预览
- 选中的图标和颜色应用到物模型

## 十、注意事项

### 10.1 组件设计原则

- 单一职责：每个组件只负责一个功能
- 可复用性：通用组件应设计为可配置
- 可维护性：组件结构清晰，命名规范

### 10.2 性能优化建议

- 使用 React.memo 避免不必要的重渲染
- 大列表使用虚拟滚动
- 图片使用懒加载
- 合理使用 useMemo 和 useCallback

### 10.3 无障碍设计

- 语义化 HTML 标签
- 键盘导航支持
- 适当的 ARIA 属性
- 足够的颜色对比度
